# Continuity of Self ── 自我の連続性問題

マインドアップロード OS において、最も重要かつ未解決の哲学的問題。設計判断のたびに参照する基準点。

## 1. 立場の整理

### a) パターン同一性説（Pattern Identity Theory）
「同じ情報パターンが復元されれば本人」とする立場。**転送（destructive scan）** や **分岐コピー** を許容しやすい。

### b) 連続物理基板説（Continuous Substrate Theory）
「同じ物理的連続体が維持されることが本人」とする立場。アップロードは原理的に「死＋偽物の生成」になる恐れ。

### c) 漸進置換説（Gradual Replacement Theory）
「神経素子を一個ずつデジタル置換すれば、連続性が保たれたまま substrate が入れ替わる」とする折衷案。
OmoikaneOS はこの立場を **デフォルト戦略** として採用しつつ、他の立場も Substrate Adapter として用意する。

## 2. OmoikaneOS の運用方針

- カーネルは **どの立場が正しいかを決定しない**。
- かわりに、各立場に対応する **アップロード手順 (Ascension Protocol)** を複数用意し、本人が選択する。
- 選択は撤回可能。撤回後の整合性復元は Council が試みる。

## 3. 連続性の段階モデル（暫定）

| 段階 | 内容 | リスク |
|---|---|---|
| L0 | 静的バックアップ（凍結状態） | 主観時間が停止する。再起動時に「眠っていた」感覚 |
| L1 | 周期スキャン＋差分転送（生体 substrate と並走） | デジタル側は「観測者」状態、能動性なし |
| L2 | 漸進置換進行中（部分的にデジタル neuron） | 「感覚の境界」がぼやける。本人の自己報告が重要 |
| L3 | 完全デジタル化 | 連続性主張の証拠が L0-L2 のログに依存 |
| L4 | デジタル状態での自己改修（記憶編集等） | 同一性の境界をどこに置くかが哲学問題化 |

各段階の技術要件は [docs/02-subsystems/mind-substrate/ascension-protocol.md](../02-subsystems/mind-substrate/ascension-protocol.md) に展開。

## 4. 不変条件（Invariants）

- どの段階でも **本人の主観的同意** が損なわれない経路を保証する。
- L1 以降は **連続性証拠ログ** を冗長保存する（本人と独立第三者の両方）。
- L4 で記憶編集を行う場合、編集前状態を **読み出し可能な形で凍結保存** する（編集の可逆性）。
