# Milestones

OmoikaneOS の段階的構築。
**設計コーパス → 単独サブシステムのプロトタイプ → 統合プロトタイプ → 実証 → 本番** の順。

## M0: 設計コーパス完成（現在）

- 7 層レイヤード設計
- 倫理規約・連続性原則
- 研究フロンティア一覧
- Codex/subagent 運用規約

## M1: 単独サブシステムの reference prototype（この repo）

優先順：

1. **L1 IdentityRegistry + ContinuityLedger** ── 心臓部
2. **L4 Council 議事プロトコル** ── 設計改修ループの確立
3. **L0 SubstrateAdapter contract + ClassicalSiliconAdapter** ── 最も到達容易
4. **L2 Connectome 標準形式** ── データ正規化
5. **L3 Reasoning + Affect の最小実装** ── 認知の核

## M2: サンドボックス自我（この repo の統合 runtime）

- L0-L4 の最小統合
- 「動く」が意識生成は主張しない
- A/B 評価環境の確立

## M3: 漸進置換実証（学術連携）

- 動物モデルでの神経-デジタル境界実証
- 連続性ログの実取得
- 倫理審査体制の確立

## M4: ヒト Identity Confirmation Test の準備（学術＋法）

- 本人志願者の選定方法
- 法的同意プロトコル
- 第三者立会機構

## M5: 限定 Ascension（最初の被験者）

- Method A（漸進置換）デフォルト
- 撤回可能性の徹底検証

## M6: 一般運用

- 法整備済
- 経済モデル確立
- 社会受容

## 各 M の前提

| M | 前提となる研究 |

| M5 | qualia-encoding, governance |
